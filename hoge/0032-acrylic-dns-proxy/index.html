    Acrylic DNS ProxyをLAN用のDNSとして使う

    [質問]

    Windowsに"Acrylic DNS Proxy"を入れて、LAN内にあるAndroidスマホから、LAN内のAndroidタブレットをホスト名でアクセスする手順を教えてください。

    [回答] Gemini

    Windows PCをDNSサーバーとして活用し、Android端末同士をホスト名で通信させる設定ですね。少しマニアックな構成ですが、一度設定してしまえばIPアドレスを覚える必要がなくなるので非常に便利です。

    以下の手順で進めていきましょう。

    1. Windowsでの準備（Acrylic DNS Proxyの設定）

    まずは、Windows PC（192.168.x.x 等の固定IP推奨）にAcrylicをインストールします。

    1. インストール: 公式サイトからダウンロードしてインストールします。
    2. 設定ファイルの編集: スタートメニューから Edit Acrylic Configuration を開き、以下を確認・変更します。

    3. Address: 0.0.0.0 になっていることを確認（LAN内からの要求を受け付けるため）。

    4. ホスト名の登録: Edit Acrylic Hosts を開き、アクセスしたいAndroidタブレットの情報を追記します。 

      # IPアドレス [スペース] ホスト名
192.168.1.50  my-tablet.lan

    Note: AndroidタブレットのIPアドレスは、ルーターの設定などで必ず固定しておいてください。

    1. サービスの再起動: Restart Acrylic Service を実行して設定を反映させます。

    2. Windowsファイアウォールの解放

    外部（Androidスマホ）からのDNSリクエストを許可する必要があります。

    • 受信規則の設定: UDPの 53番ポート を許可してください。
    • Windowsの「セキュリティが強化されたWindows Defenderファイアウォール」から、新しい規則を作成し、「ポート」→「UDP」→「53」を指定して接続を許可します。

    3. Androidスマホ側の設定

    次に、アクセス元となるAndroidスマホのWi-Fi設定を変更します。

    1. Wi-Fi設定を開く: 接続中のSSIDを長押し、または歯車アイコンをタップします。
    2. ネットワークの編集: 「IP設定」を「静的（Static）」に変更します。
    3. DNSサーバーの指定:
    4. DNS 1: Windows PCのIPアドレスを入力します。

    5. DNS 2: 8.8.8.8（予備としてGoogleのDNSなどを入れておくと安心です）。

    6. 保存: 設定を保存します。

    4. 接続確認

    Androidスマホのブラウザやアプリから、設定したホスト名でアクセスしてみましょう。

    • スマホのブラウザに http://my-tablet.lan と入力して、タブレットで動いているサービス（サーバー等）が表示されるか確認してください。
    • うまくいかない場合は、スマホ側で一度Wi-Fiをオフ/オンにすると、DNSキャッシュがクリアされて繋がることがあります。

    注意点：Androidの「プライベートDNS」

    最近のAndroidには「プライベートDNS」という設定項目がありますが、これは通常「オフ」にしておいてください。これが有効だと、せっかく設定したローカルのDNS（Windows PC）を無視して外部の暗号化DNSに飛んでしまうことがあります。

    設定は無事に完了しそうでしょうか？もし特定のアプリで繋がらないなどあれば、詳細を教えてくださいね。
