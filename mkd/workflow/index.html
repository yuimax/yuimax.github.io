基本的な処理の流れ

Web サイトを GitHub Pages で公開するまでの流れ。

本サイトの作成過程を例にする。

1. プロジェクト名を決める

目標は、GitHub ページに「MkDocsの使い方」というサイトを作ること 。

プロジェクト名を mkd とする。

2. プロジェクトの新規作成

mkdocs new コマンドで、プロジェクトを新規作成する。

mkdocs new mkd
↓
INFO    -  Creating project directory: mkd
INFO    -  Writing config file: mkd\mkdocs.yml
INFO    -  Writing initial docs: mkd\docs\index.md

作成されたディレクトリに移動する。

cd mkd

3. ローカルサーバーを起動する

MkDocs のイト作成は、ローカル Web サーバー上で 実際にサイトの表示や動きを確認しながら進めていく。

まず mkdocs serve コマンドでサーバーを起動する。

mkdocs serve
↓
INFO    -  Building documentation...
INFO    -  Cleaning site directory
INFO    -  Documentation built in 0.13 seconds
INFO    -  [23:48:06] Watching paths for changes: 'docs', 'mkdocs.yml'
INFO    -  [23:48:06] Serving on http://127.0.0.1:8000/

サーバー稼働中はキーボードは反応しないが [Ctrl]+[C] で止めることはできる。

サーバーが動くとブラウザで http://localhost:8000/ にアクセスできる。 最初はデフォルトのトップページ「Welcome to MkDocs」が表示されるはずだ。
